title: "Menu Configuration"

---

Menus are custom inventory GUIs that players can interact with. They're defined in YAML files in the `menus/` folder.

## Menu Structure

Each menu file contains:

<ParamField path="title" type="string" required>
  The displayed title of the inventory. Supports color codes and hex colors.
</ParamField>

<ParamField path="size" type="integer" required>
  The number of slots in the inventory. Must be a multiple of 9 (9, 18, 27, 36, 45, 54).
</ParamField>

<ParamField path="items" type="object" required>
  A map of items to display in the menu.
</ParamField>

## Item Properties

Each item in the menu can have:

<ParamField path="material" type="material" required>
  The Minecraft material for the item.
</ParamField>

<ParamField path="name" type="string">
  Display name of the item. Supports color codes and hex colors.
</ParamField>

<ParamField path="slot" type="integer" required>
  The slot position (0-based index). Use `-1` for fill items.
</ParamField>

<ParamField path="lore" type="array">
  List of lore lines.
  
  ```yaml
  lore:
    - "&7Line 1"
    - "&7Line 2"
  ```
</ParamField>

<ParamField path="amount" type="integer" default="1">
  Stack size of the item (1-64).
</ParamField>

<ParamField path="actions" type="array">
  Actions to execute when the item is clicked.
</ParamField>

## Player Heads in Menus

Menus support three types of player heads:

<Tabs>
  <Tab title="URL Texture">
    Use a texture URL from minecraft.net:
    
    ```yaml
    creative:
      material: PLAYER_HEAD
      url: http://textures.minecraft.net/texture/bcf2105bb737638833033dd8244071e75870e2e11c2617e542e8924fb2b90180
      name: "&cCreative Server"
      slot: 14
    ```
  </Tab>
  
  <Tab title="Player Name">
    Use a specific player's skin:
    
    ```yaml
    player_head:
      material: PLAYER_HEAD
      texture: player
      name: "Player Head"
      slot: 0
    ```
  </Tab>
  
  <Tab title="HeadDatabase">
    Use HeadDatabase plugin IDs (requires HeadDatabase):
    
    ```yaml
    hdb_head:
      material: PLAYER_HEAD
      hdb: 12345
      name: "HDB Head"
      slot: 0
    ```
  </Tab>
</Tabs>

## Leather Armor Colors

Customize leather armor colors using hex codes:

```yaml
factions:
  material: LEATHER_CHESTPLATE
  color: 8ad592  # Without the # symbol
  name: "&cFactions Server"
  slot: 22
```

<Note>
  Omit the `#` symbol when specifying hex colors for leather armor.
</Note>

## Custom Model Data

For resource pack support, you can specify custom model data:

```yaml
survival:
  material: APPLE
  model_data: 10001
  name: "<#CCFFDD>Survival Server"
  slot: 12
```

## Menu Actions

<Tabs>
  <Tab title="BungeeCord">
    Connect player to another server (requires BungeeCord):
    
    ```yaml
    actions:
      - "[BUNGEE] survival"
    ```
  </Tab>
  
  <Tab title="Close Menu">
    Close the current menu:
    
    ```yaml
    actions:
      - "[CLOSE]"
    ```
  </Tab>
  
  <Tab title="Player Command">
    Execute command as player:
    
    ```yaml
    actions:
      - "[PLAYER] hubbly movement none"
      - "[CLOSE]"
    ```
  </Tab>
  
  <Tab title="Link/URL">
    Open a clickable link in chat:
    
    ```yaml
    actions:
      - "[LINK] Click me for discord;Discord link;discord.com/invite"
    ```
    
    Format: `[LINK] <hover_text>;<link_text>;<url>`
  </Tab>
  
  <Tab title="Menu">
    Open another menu:
    
    ```yaml
    actions:
      - "[MENU] selector"
    ```
  </Tab>
</Tabs>

## Fill Items

Fill empty slots with a specific item using `slot: -1`:

```yaml
fill:
  slot: -1
  material: BLACK_STAINED_GLASS_PANE
  name: " "  # Empty name
  actions:
    - "[CLOSE]"
```

## Opening Menus

### Via Command

```bash
/hubbly menu <menu_name>
```

Example:
```bash
/hubbly menu selector
/hubbly menu socials
```

### Via Item Action

In `items.yml`:
```yaml
compass:
  name: "&cCompass"
  material: COMPASS
  actions:
    - "[MENU] selector"
```

### Via Menu Action

Chain menus together:
```yaml
back_button:
  material: ARROW
  name: "&cBack"
  slot: 0
  actions:
    - "[MENU] main"
```

## Tips and Best Practices

<Warning>
  - Menu files must be placed in the `menus/` folder
  - File names should use lowercase and no spaces (use underscores)
  - Size must be a multiple of 9 (max 54)
  - Slots are 0-based (0 = first slot, 8 = last slot in first row)
</Warning>

<Note>
  **Slot Layout Reference:**
  
  9-slot menu: 0-8
  
  18-slot menu: 0-17
  
  27-slot menu: 0-26 (3 rows)
  
  36-slot menu: 0-35 (4 rows)
  
  45-slot menu: 0-44 (5 rows)
  
  54-slot menu: 0-53 (6 rows - max)
</Note>

<Accordion title="Design Tips">
  - Use fill items to prevent clicking empty slots
  - Space out important items for better visual clarity
  - Use consistent naming conventions across menus
  - Add descriptive lore to explain what each option does
  - Test menus with `/hubbly menu <name>` before deployment
  - Use color gradients for professional-looking titles
</Accordion>
